{# MFA enrollment nag banner for owner-role users without MFA enabled. Rendered from base.html when the ``mfa_nag_visible`` template variable is truthy. The visibility flag is computed by the ``inject_mfa_nag_visible`` context processor in ``app/__init__.py``; see audit finding F-095 / commit C-12 of the 2026-04-15 security remediation plan. The banner is dismissible (Bootstrap ``data-bs-dismiss="alert"``) for the current page view only. Clicking the close button removes the banner from the DOM but persists no server-side state -- the banner reappears on the next navigation until the owner enrols a TOTP authenticator at ``/mfa/setup`` and confirms it at ``/mfa/confirm``. This per-page-load dismissal is intentional: a persistent dismissal would let the owner permanently silence a security control the audit finding rates as worth the friction. Companions are excluded by the context processor (the audit scopes the nag to owner-role users, who are de facto administrators in this app). Anonymous users never see the banner because the processor short-circuits on ``current_user.is_authenticated``. #}
