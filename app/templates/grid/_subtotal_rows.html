{# Summary subtotal rows -- the two self-refreshing sections. Returned by the grid.subtotal_rows HTMX endpoint and also included inline by grid/grid.html for the initial render. The grid keeps Total Income grouped with the income block and Total Expenses + Net Cash Flow grouped with the expense block, so the two groups live in two separate elements at different positions in the table. A may hold multiple elements, so wrapping each group in its own (rather than tagging bare elements with hx-swap-oob, whose table-row parsing is unreliable) lets htmx swap a whole, parseable table section. ONE GET per balanceChanged: only the income carries the self-refresh hx-get / hx-trigger / hx-swap="outerHTML"; the endpoint response also carries the expense as an hx-swap-oob fragment (a whole is a parseable OOB target, unlike a bare ), so the single GET refreshes both sections. The expense therefore has the matching id but no trigger of its own. Two render modes: - Endpoint response (``oob`` true): emit BOTH sections -- income (with the self-refresh trigger) and expense (as an hx-swap-oob fragment) -- so one GET refreshes both. - Initial page-load include (``oob`` false): grid.html includes this once per section via the ``section`` variable, so each lands at its own position in the table. The expense is emitted as a plain section here (no hx-swap-oob), because the OOB target does not exist until the page is built. The figures are computed in the route with Decimal and passed in via ``subtotals``; this template only displays them. Expected context (* required): periods * -- the visible PayPeriod slice. subtotals * -- dict keyed by period.id, each with .income / .expense / .net Decimals. num_periods -- visible column count, for the self-refresh url. start_offset -- window offset, for the self-refresh url. account -- the grid Account (None for the zero-accounts edge), for the self-refresh url's account_id. oob -- (optional) bool, default false. True for the grid.subtotal_rows HTMX response (emit both sections, expense as an OOB fragment). section -- (include-only) "income" or "expense": which single to render on the initial page-load include.
