{# Mobile "This Period" partial -- single-period mobile panel. Renders ONE pay period (defaulting to today's period when `start_offset == 0`) as a compact mobile panel: period-nav header with prev/next arrow links, collapsible income and expense cards, an optional net cash flow bar, and an optional projected balance card. The arrow links use the same `?periods=N&offset=M` URL convention as the desktop selector at `grid/grid.html` and carry a `#this-period` fragment so the page lands back on this tab after the full GET; the matching hash-routing handler lives in `app/static/js/mobile_grid.js`. Used by: - the "This Period" tab in `grid/_mobile_grid.html` (owner). - the companion route in Commit 13 (shared partial per design decision D-B in the mobile-first v3 plan). Expected Jinja context (* required): periods * -- list[PayPeriod]; this partial uses `periods[0]` (the leftmost visible period; equals `current_period` when `start_offset == 0`). current_period * -- PayPeriod; the period containing today. Anchors the offsets emitted by the jump-to . start_offset * -- int; offset of `periods[0]` relative to `current_period.period_index`. Drives the prev / next arrow hrefs. income_row_keys * -- list[RowKey] in render order. expense_row_keys * -- list[RowKey] in render order. matched_by_row_period * -- precomputed `(category_id, template_id, txn_name, period_id) -> list[Transaction]` dict from the grid route (Phase 1, Commit 2). entry_sums * -- dict[txn_id -> entry summary]; consumed by `render_row_card` for the progress-bar render path on Projected envelopes with entries. entry_lists -- (optional) dict from `entry_service.build_entry_lists_dict` mapping envelope txn_id -> {entries, remaining, out_of_period_ids}. Forwarded to `render_row_card` so the inline `_transaction_entries.html` renders server-side instead of the per-card HTMX `hx-trigger="load"` fan-out that blew past the 30-per-minute rate-limit ceiling. When undefined or missing the txn's id, the inline entries section is skipped (matching the non-envelope render path). today -- (optional) `datetime.date` used as the default value of the add-entry form's hidden `entry_date` input inside `_transaction_entries.html`. Only consumed indirectly through `render_row_card`; required whenever `entry_lists` is provided. subtotals -- (optional) dict[period_id -> PeriodSubtotal]. When undefined (companion contract per Section 12 Q-2 of the v3 plan), the income / expense card-header totals and the Net Cash Flow bar are gracefully omitted; row cards still render. balances -- (optional) dict[period_id -> Decimal]. When undefined or missing the key for `period.id`, the Projected Balance card is gracefully omitted. can_edit -- (optional) bool, default True; forwarded to `render_row_card` so companion (`can_edit=False`) drops the bottom-sheet `data-mobile-*` attributes. show_period_nav -- (optional) bool, default True. When False, the partial omits the prev/next-arrow header and the jump-to -- Native picker lets the user reach any non-adjacent period in one tap instead of N taps on `[<]` / `[>]`. Method=GET so the hidden `periods=1` + `offset=` land in the same URL shape as the arrow links and the desktop selector at `grid/grid.html:24-49`. CSRF is not required for GET per CLAUDE.md "HTMX Patterns / State-changing actions must use POST" (this is read-only navigation, no state change). Submission is wired in `app/static/js/mobile_grid.js` via a delegated `change` listener (CSP-friendly per CLAUDE.md "No inline scripts"). #}
