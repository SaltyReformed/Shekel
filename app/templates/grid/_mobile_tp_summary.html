The companion route passes no ``subtotals`` / ``balances`` (Q-2 (c) of the mobile-first v3 plan), so the ``{% if subtotals is defined %}`` guard makes this partial render nothing on the companion page -- there the card swap alone is the whole update. Why it self-refreshes: the mobile / companion Mark Paid path swaps a single card in place and fires ``HX-Trigger: mobileCardSettled`` instead of the desktop ``gridRefresh`` (which reloads the whole page). Settling a transaction changes the projected balance and the period subtotals, so without a refresh these numbers would go stale. The root element below mirrors the desktop ``_balance_row.html`` self-refresh convention: ``hx-get`` its own route on ``mobileCardSettled from:body`` and ``outerHTML``-swap itself, which re-arms the trigger each cycle. All four owner-mobile money figures stay correct on one request: - Net Cash Flow and Projected Balance render inline here (the primary swap body). - The Income and Expense section-header totals live in ``_mobile_this_period.html`` (inside the collapsible card headers). This partial refreshes them via ``hx-swap-oob`` fragments keyed to their span ids -- emitted ONLY when the route sets ``oob`` true, never on the initial page-load include (where HTMX ignores ``hx-swap-oob`` and the duplicate spans would otherwise double the totals on screen).
