# Cloudflare Tunnel configuration.
# cloudflared runs on the 'homelab' Docker network and reaches services
# by their Docker compose service name. Traffic arrives from Cloudflare's

# HTTP over the internal Docker network.

tunnel: ""

  - hostname: ""
    service: http://jellyfin:8096

  - hostname: shekel.saltyreformed.com

# Auditor observations (recorded here for cross-reference; full analysis in
# docs/audits/security-2026-04-15/reports/13-attack-surface.md):
#
# 1. No Cloudflare Access (Zero Trust) policies configured (confirmed by developer).
#    The only auth gate between the public internet and any Shekel route is
#    Flask's login_required decorator -- cloudflared passes traffic through.
#
# 2. shekel.saltyreformed.com routes directly to shekel-prod-app:8000 (Gunicorn),
#    bypassing BOTH the bundled shekel-prod-nginx (disabled via override profile)
#    AND the shared /opt/docker/nginx reverse proxy.  None of the Nginx-layer
#    limits (client_max_body_size 5m, timeouts, security headers, real_ip
#    processing) apply to WAN traffic.
#
# 3. originRequest lacks noTLSVerify override for shekel/immich (scheme is
#    http:// so TLS is not involved on the origin side -- cloudflared terminates
#    TLS at the edge and sends plain HTTP over the homelab bridge network).
#
# 4. The catch-all entry "service: http_status:404" is correctly the last rule.
#    Cloudflared evaluates ingress top-to-bottom; first match wins.
